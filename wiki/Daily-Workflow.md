# Daily Workflow

This is the operator's view of what runs every day, in order. Once the system is configured, the only required action is reading Telegram.

---

## 9:00 AM — launchd fires automatically

A launchd plist at `~/Library/LaunchAgents/com.animesh.linkedin-agent.daily.plist` triggers `agent run-daily` at 9am without any manual intervention. The process logs to `logs/launchd.log`.

What happens inside:

1. `truststore.inject_into_ssl()` patches the SSL stack to handle the corporate certificate chain.
2. The `IdeatorAgent` fires first. It calls `fetch_and_store()` in `src/inputs/news.py` to pull the latest items from all five RSS feeds (ET Tech, YourStory, Inc42, TechCrunch AI, Hacker News), deduplicates by URL hash, and stores new rows in the `news_items` table.
3. The Ideator then reads up to 40 news items from the last 48 hours, plus the 15 most recent non-rejected posts from the `posts` table (for theme-avoidance signal), and ranks 5–10 post angles by relevance score. The top angle is selected automatically.
4. The `WriterAgent` takes the top angle and generates 2 draft variants using `claude-opus-4-7`. Both drafts land in the `posts` table with `status="draft"`.
5. The `CriticAgent` scores each draft independently using `claude-haiku-4-5-20251001` at temperature 0.0 (deterministic). Each score is four integers (1–10) on hook strength, voice match, argument quality, and hygiene. The overall is their mean.
6. Both scored drafts are sent to Telegram.

---

## ~9:20 AM — 2 Telegram messages arrive

Each message follows this format exactly:

```
── DRAFT 1 [story] ──
Score 7.8/10 · Hook 8 · Voice 7 · Sub 8 · Clean 8
"Hook is punchy. Voice drifts slightly in paragraph 3."

The first time I shipped a feature that I knew was wrong...
[full post body]

[✅ Approve]  [✏️ Edit]  [❌ Reject]
```

The score line is generated by `_score_line()` in `src/outputs/telegram_bot.py`. Fields are:
- **Hook** — `hook_strength`
- **Voice** — `voice_match`
- **Sub** — `argument_quality` (substance/argument quality)
- **Clean** — `hygiene` (LinkedIn format, no forbidden phrases, clean ending)

The quoted line below the score is the Critic's `verdict` — a one-sentence qualitative read on the draft.

## Approval flow

**Tap ✅ Approve**

The bot calls `update_post_status(db_path, post_id, "approved")`, strips the inline keyboard, and sends a confirmation. It then calls `schedule_post()` in `src/outputs/typefully.py`, which posts to `https://api.typefully.com/v2/social-sets/{id}/drafts` with `"publish_at": "next-free-slot"`. On success, you get a follow-up message with the Typefully draft ID.

Total time: under 5 seconds from tap to scheduled.

---

## Edit flow

**Tap ✏️ Edit**

The bot replies "Send me your edited version." and sets `_pending[chat_id] = ("awaiting_edit", post_id)`. Send the edited post body as a plain Telegram message. The bot calls `update_post_body()` then `update_post_status(..., "approved")` atomically, confirms "Saved and approved," and immediately schedules to Typefully.

One detail: if you accidentally paste the full Telegram message including the score header, `_strip_draft_header()` removes everything before the first blank line so only the post body is saved.

---

## Rejection flow

**Tap ❌ Reject**

The bot prompts "Rejection reason? (or send 'skip')". Send a reason or the literal word `skip`. Either way the post is marked `status="rejected"` in the DB. The reason is logged via structlog but not currently stored in the schema — it lives in `logs/agent.log` only.

Friction is intentional: one tap to initiate, one message to complete, nothing else required.

## Manual override — running with a specific idea

At any time (not just 9am), you can trigger the pipeline with a specific idea:

```bash
uv run --system-certs python scripts/run_daily.py --idea "why most PM prioritization frameworks fail in practice"
```

The `AngleMappingAgent` maps the free text to a `topic_lane`, `sub_topic`, and `context_note` using `claude-haiku-4-5-20251001`, echoes the mapping to the terminal, then proceeds through the same Writer → Critic → Telegram pipeline.

If you already know the exact lane and sub-topic:

```bash
uv run --system-certs python scripts/run_daily.py --topic pm_craft --sub-topic prioritization --context "focus on frameworks that break down at scale"
```

The equivalent CLI alias (via `src/cli.py`):

```bash
uv run --system-certs agent run-daily --idea "..."
```

## Checking logs

Tail the launchd output log in real time:

```bash
tail -f logs/launchd.log
```

The structured JSON log at `logs/agent.log` has an entry for every LLM call: `model`, `input_hash`, `output_hash`, `input_tokens`, `output_tokens`, `latency_ms`. grep by event name:

```bash
grep "llm_call" logs/agent.log | jq '{model, input_tokens, output_tokens, latency_ms}'
grep "draft_approved" logs/agent.log
grep "typefully_scheduled" logs/agent.log
```

## Killing a hung bot

If launchd tries to start a new run while yesterday's bot is still polling (Telegram will return a `Conflict` error — only one polling instance per bot token), kill the stale process:

```bash
pkill -f "agent run-daily"
```

Then re-trigger manually or wait for tomorrow's scheduled run. The launchd plist will start a fresh process at 9am regardless.
